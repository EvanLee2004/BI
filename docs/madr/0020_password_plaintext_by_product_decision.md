# 0020 · 管理员可见密码：明文存储（产品拍板）

- **状态**：Accepted
- **日期**：2026-07-20
- **决策者**：李明昊
- **相关**：Supersedes [0019_password_hashing_pbkdf2.md](./0019_password_hashing_pbkdf2.md)

## 背景

任务书63·H-05 将 `看板账号.json` 改为 PBKDF2 哈希存储，管理端仅能「重置」、不可查看现有密码。  
产品使用场景：财务内网看板，管理员（最高权限）需要代查/代改同事看板口令，**可查看全部账号密码是硬需求**——与哈希数学互斥。

## 决策

**恢复明文为真相源**（`密码` 字段），`/api/accounts` 管理员会话下发明文；管理端账号表密码列可见可编辑。

已知并接受：文件可读则口令可读。缓解措施（本单一并落地）：

| 缓解 | 说明 |
|------|------|
| 文件 `chmod 0o600` | 账号/智云凭据/管理员密钥统一私密写盘 |
| 不进 git | 真实 `数据/看板账号.json` 仅在部署机 |
| 纯内网 | 无公网暴露管理端 |
| 登录防爆破 | login_guard |
| 会话 12h + 改密踢会话 | SESSION_TTL + 密码版本 |
| 审计不记明文 | 只记「改密码」 |

## 不做什么

- 不恢复 PBKDF2 落盘
- 不要求 HTTPS（另议）
- 不把同事口令写进日志/审计摘要

## 后果

- 生产若从未上过 63 哈希版 → **零迁移**（本单先于 push）
- 若某处已是哈希-only 行 → 该行不可登录直至管理员写入明文
