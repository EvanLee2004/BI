# v1.4.0-beta 前后端分离 · 交付说明

基准：`efacf8c` / v1.3.1-beta。原则：**搬家不装修**（无 React、无 npm）。

## 三个问题（分离的好处）

### 1. 只改前端、不碰 Python 算账

| 要改什么 | 动哪个文件 | 不动什么 |
|---|---|---|
| 暗色主色 / 间距 / 圆角 | `static/css/theme.css` | profit / ingest / db |
| 周期切换动画 / 回款高亮 class | `static/js/cockpit.js` | 同上 |
| BU 页本地「其余」弹窗 | `static/js/cockpit-bu.js` | 同上 |

例：把 KPI 数字字号从 26px 调到 28px → 只改 `theme.css` 里 `.kpi-val`（或对应类），**不必改 `profit.py`，也不必重算库**。

### 2. 全部 JSON API 及外部复用

| API | 用途 |
|---|---|
| `GET /api/v1/session` | 当前登录态 |
| `POST /api/v1/login` / `logout` | JSON 登录登出 |
| `GET /api/v1/cockpit` | 整体驾驶舱 JSON（含 `numbers` 全量关键数字） |
| `GET /api/v1/cockpit/bu/{name}` | 单 BU JSON |
| `GET /api/v1/cockpit/view` | 像素级 HTML（原 render 输出 + 外置 CSS/JS） |
| 既有 `/api/manual*` `/api/detail` `/api/daily` … | 管理端与动态查询，行为同 v1.3.1 |

飞书机器人示例：内网会话下 `GET /api/v1/cockpit` → 读 `numbers.periods["2026年"].kpi.pretax_profit` 推送，**不要刮 HTML**。

### 3. 切周期 / 切 BU 的加载方式

| 操作 | 基准版 v1.3.1 | v1.4 |
|---|---|---|
| 切年/季/月 | 前端只显示/隐藏已预渲染的 `.pv` 块，**不重算、不重拉** | **相同**（逻辑在 `static/js/cockpit.js`，内容未改） |
| 首次打开整体页 | 服务端一次吐完整 HTML（CSS/JS 内联） | 壳 `shell.html` → `fetch /api/v1/cockpit/view` → 同一套 HTML（CSS/JS 走 `/static` 可缓存） |
| 切 BU | 整页导航 `/bu/{name}` | **相同**（仍整页；JSON 可另用 `/api/v1/cockpit/bu/{name}`） |

结论：周期切换**本来就不是整页重渲**；分离后把资源外置 + 增加 JSON，便于缓存与外部系统，**观感与数字对齐 golden**。

## 回退
