## 批次 A · 做了什么 × 证据

### A1 原子化重建
- **现象→证据→缺陷**：`reset_std_tables` 先 commit 再插 → 中途崩溃 std 全空。
- **修法**：`BEGIN IMMEDIATE` → DELETE+INSERT → 一次 COMMIT；失败 ROLLBACK。
- **测试**：`tests/test_db_atomic_rebuild.py`（第 3 表抛异常 → 旧行完整）。

### A2 WAL / 忙等
- **修法**：`db.connect` 统一 `journal_mode=WAL`、`busy_timeout=5000`、`synchronous=NORMAL`。
- **测试**：重建未提交期间另一连接可读旧 std。

### A3 金额整数分
- **范围**：5 张 std 金额列 + manual 手填/BU/历史/预算/预算历史 → INTEGER **分**。
- **进料**：`money.yuan_to_fen`（`Decimal` + `ROUND_HALF_UP`）；禁止 float×100 直乘。
- **算账**：profit 仍吃 **元 float**（load 时分→元）；显示格式化不变。
- **定位键**：normalize 仍用 **元** 算哈希再转分入库 → **存量 adj 定位键无需重映射**。
- **adj 原值/新值**：保持 **元 TEXT**；重放时分↔元换算。
- **迁移**：`schema.SCHEMA_VERSION=2`，`migrate_money_to_fen_if_needed` 幂等；迁移前 `看板.db.bak-fen-*`。
- **验收**：回归红线 32 周期通过；diff 改为递归 tuple + **半分（0.005 元）** 容差（文件路径 float 噪声 vs 分读回）。
- **测试**：`tests/test_money_fen.py`（0.005/负/超大/空/迁移/调整）。

### A4 定位键重复
- **实际行为（文档化）**：
  - **写调整**：命中 >1 行 → **ValueError 拒绝**（不静默改多行）。
  - **重放**：命中 >1 行 → **过期疑似**、不套用、体检黄。
- **审计**：`db.audit_duplicate_locators` → 管道报告；有重复 → 黄。
- **测试**：`tests/test_db_a4a7.py` + 既有 `test_adjust.TestLocatorCollisionGuard`。

### A5 规模基准（只记录）
见 `docs/bench_scale_100k_latest.json`（合成 10 万收入行 + 1 万费用行，本机 2026-07-16）：

| 指标 | 值 |
|------|-----|
| 种子写入 | ~0.35 s |
| query_detail 首页 50 行 | ~4 ms |
| 关键词搜索 | ~32 ms |
| db 文件 | ~10.2 MB |
| 峰值 RSS | ~38 MB |
| quick_check | ok |

**结论**：无超线性异常；**未加索引**（EXPLAIN 未必要；翻页/搜索可用）。完整 xlsx 进料 + generate 未在本基准重跑（依赖真实进料口体积）。
