# 经营驾驶舱 · Windows 部署手册（v7.3 · 2026-07-10 重写）

> 目标机器：财务部 Windows 电脑（内网、常开为佳），建议装到 `D:\看板\看板正式程序\`（路径不要带空格）。
> 部署形态：**FastAPI 常驻服务（双端网页）+ 每日计划任务自动更新 + 智云在线抓数（账号密码自动登录）+ 共享盘拉收单台账**。
> 旧 v6 手册（纯静态 HTML + Caddy 方案）已作废，以本文为准。导出 zip 兜底、台账公式缓存两条老边界仍有效（见"七"）。

---

## 一、装环境（只做一次，约 20 分钟，需要联网）

1. **装 Python 3.10+**：https://www.python.org/downloads/ ，安装时**务必勾选 "Add python.exe to PATH"**。
2. **拿代码**：把整个 `看板正式程序` 文件夹拷到 `D:\看板\`（U盘/内网传输均可）。**不要拷 macOS 上的 `.venv` 文件夹**——虚拟环境不能跨系统用，带过来了就删掉重建。
3. **建虚拟环境 + 装依赖**：在 `D:\看板\看板正式程序\` 打开 cmd：
   ```
   python -m venv .venv
   .venv\Scripts\pip install -r requirements.txt
   .venv\Scripts\pip install playwright
   .venv\Scripts\playwright install chromium
   ```
   最后一步下载无头浏览器（约 150MB，智云自动登录用），**必须做**。

## 二、填配置（只做一次）

1. **`数据\智云配置.json`**（不进 git，手工放到部署机）：从开发机拿模板，核对 `base_url`（内网地址）与四张表 `worksheetId`（模板已填好）；`username`/`password` 可以先留测试账号——**推荐部署完在管理员端 → 设置 → 智云账号里填正式账号**，保存后点「立即更新」即生效，账号内部 ID 程序自动获取。
2. **`config.json` 的 `ledger_share_path`**：把占位符改成收单台账真实共享盘路径（形如 `\\\\<服务器IP>\\财务部\\...\\收单台账.xlsx`，JSON 里反斜杠写双份）。⚠ 此改动**只留在部署机本地，绝不提交 git**。
3. **管理员密码**：首次启动自动生成 `数据\管理员密钥.json`，默认口令见 README；建议部署时设环境变量 `KANBAN_ADMIN_PW` 换掉默认口令。

## 三、首跑验证（跑通一次再挂服务）

```
.venv\Scripts\python run.py
```
预期：收单台账 fetched（共享盘可达）、智云 下单/回款/项目明细 fetched（**内部译员在换专用账号前是 local_fallback，属正常**）、生成 `output\经营驾驶舱.html`。首跑约 2~3 分钟（在线抓 80 秒 + 32 个周期计算约 1 分钟）。

## 四、起服务 + 定时更新

1. **起服务**：双击 `启动看板服务.bat`（内部执行 `run.py --serve`）。
   - 用户端（老板看）：`http://<本机IP>:8018/`；管理员端：`http://<本机IP>:8018/admin`
   - 同 WiFi 手机可直接访问；端口可用环境变量 `KANBAN_PORT` 改。
   - 防火墙首次询问选"专用网络允许"；域环境可能需 IT 开 8018 入站规则。
2. **每日自动更新**：右键**以管理员身份**运行 `注册每日更新.bat`（注册计划任务，默认 09:30 跑 `run.py --scheduled`）。之后改时间在**管理员端 → 设置**里改，不用重跑 bat。
3. **开机自启（建议）**：把 `启动看板服务.bat` 快捷方式放进 `shell:startup`（Win+R 输入），重启后服务自动拉起。

## 五、部署机必做复验清单（Windows 特有，开发机验不了）

- [ ] `run.py` 首跑：共享盘 fetched + 智云三源 fetched
- [ ] `--serve` 起来后点管理员端「立即更新」→ 服务进程内完整跑一次在线抓（**Playwright 在 serve 进程内登录，开发机未实测，必须验**）
- [ ] `注册每日更新.bat` 注册成功：`schtasks /Query /TN "经营驾驶舱每日更新"` 能查到；次日确认真跑了
- [ ] 管理员端设置页改「每日更新时间」→ 计划任务时间真的变了（`schtasks /Query` 复核）
- [ ] 手机（同 WiFi）打开用户端正常；管理员端登录、改一行数、撤销，全链路通
- [ ] `数据\备份\` 每天多一份 `看板_YYYYMMDD.db`，超过设置天数自动删最旧

## 六、日常运维（财务只需要会这三件事）

| 事 | 怎么做 |
|---|---|
| 看数 | 浏览器开 `http://<IP>:8018/`（老板）或 `/admin`（财务） |
| 手动更新 | 管理员端顶栏「立即更新」（约 1~2 分钟，按钮显示进度） |
| 改设置 | 管理员端「设置」：每日更新时间 / 备份保留天数 / 智云账号 |

抓不到数不会报错中断——自动用上次数据继续算，顶栏体检变黄，点开看原因（不在内网 / 账号失效 / 共享盘不可达 / 权限不足）。

## 七、已知边界

- **内部译员源**需要能看「任务」表全量的账号（测试账号只有个人视图 85 行，被行数护栏挡住走人工导出文件兜底；换专用账号后自动全绿，代码零改）。
- 智云账号密码明文存部署机本地 `数据\智云配置.json`（gitignore 挡住不进库；部署机在财务掌控内，既定设计）。
- **B3**：`--scheduled` 计划任务更新库后，`--serve` 服务的内存页面不会即刻刷新——日常以管理员端「立即更新」为准；此项在 R 系列迭代中解决。
- 页面"⬇导出"=当前所选周期的整页图片（服务端截图，约2秒；需 chromium 已装——装环境第3步已含）。
- 台账公式单元格若未在 Excel 里重新保存过，openpyxl 可能读到空——台账保持纯数值粘贴习惯。
- 机器关机期间定时任务不跑、页面无法访问。
