# 甲骨易智能经营罗盘 · Windows 部署手册（v8.7 · 2026-07-12 一键更新+看门狗+多时间点+多BU账号）

> 目标机器：财务部 Windows 电脑（内网、常开为佳），建议装到 `D:\看板\看板正式程序\`（路径不要带空格）。
> 部署形态：**FastAPI 常驻服务（双端网页）+ 每日计划任务自动更新（可多时间点）+ 智云在线抓数（账号密码自动登录）+ 共享盘拉收单台账 + 管理端一键更新（看门狗重启，对标 Gitee/GitHub 远端）**。
> 旧 v6 手册（纯静态 HTML + Caddy 方案）已作废，以本文为准。（导出已改整页图片、zip 导出已删，第七节的台账公式缓存边界仍有效。）

## 一、装环境（只做一次，约 20 分钟，需要联网）

1. **装 Python 3.10+**：https://www.python.org/downloads/ ，安装时**务必勾选 "Add python.exe to PATH"**。
2. **拿代码（推荐 `git clone` from Gitee，一键更新才能用）**：装 Git（https://git-scm.com/），在 `D:\看板\` 打开 cmd：
   ```
   git clone https://gitee.com/Lee157/oracleeasy--bi.git 看板正式程序
   ```
   这样 `origin` 就是 Gitee，管理端一键更新默认对标 Gitee（国内快）。库是**公开库，clone 免密不需要令牌**。
   - **不用 git（拷文件夹）也行**，但那样没有 `.git`，一键更新用不了（只能手工换文件夹升级）；若之后想启用一键更新，在此目录 `git init` + `git remote add origin <Gitee地址>` + `git fetch` + `git reset --hard origin/main` 即可。
   - 若你是从 GitHub clone 的、又想更新走 Gitee：`git remote add gitee https://gitee.com/Lee157/oracleeasy--bi.git`，然后把 `update_remote` 设为 `"gitee"`——**写进 `数据\本地配置.json`**（`{"update_remote": "gitee"}`）**而不是改 config.json**（改 config.json 会让工作区永久脏、一键更新失效；从 Gitee clone 则默认就对标 Gitee，本条可跳过）。
   - **不要拷 macOS 上的 `.venv` 文件夹**——虚拟环境不能跨系统用，带过来了就删掉重建。
3. **建虚拟环境 + 装依赖**：在 `D:\看板\看板正式程序\` 打开 cmd：
   ```
   python -m venv .venv
   .venv\Scripts\pip install -r requirements.txt
   .venv\Scripts\pip install playwright
   .venv\Scripts\playwright install chromium
   ```
   最后一步下载无头浏览器（约 150MB，智云自动登录用），**必须做**。

## 二、填配置（只做一次）

1. **智云连接（v8.7.5 起零配置）**：服务器地址 + 四张表 `worksheetId` 已内置在程序里（`fetch_zhiyun.ZHIYUN_DEFAULTS`，clone 即有），**不用再从开发机拷 `智云配置.json` 模板**；智云换服务器/换表时在管理端「设置 → 智云账号 → 智云服务器与抓取表」里改（写本机 `数据\智云配置.json` 覆盖层）。**智云账号密码**部署完在管理员端 → 设置 → 智云账号里填，保存后点「立即更新」即生效，账号内部 ID 程序自动获取。
2. **收单台账共享盘路径（v8.7.5 起自带默认）**：config.json 出厂默认=亮晶共享盘真实路径（2026-07-13 明昊拍板进库·部署开箱即抓）。**共享盘搬家时不要改 `config.json`**——在**管理端「设置 → 智云账号·台账路径」**里改，只落 `数据\本地配置.json`（本机专属·不进 git）覆盖默认，`config.json` 保持不动。
   > ⚠ 这条是**一键更新能不能用的关键**：`config.json` 是 git 追踪文件，一旦手改它，`git status` 永久显示"有改动"，一键更新的"工作区脏就拒绝"护栏会永久生效、按钮变摆设。所以台账路径（以及更新时间、备份天数等）一律走设置页写本地覆盖文件，别碰 config.json。（`update_remote` 若要从 GitHub 改走 Gitee，也建议写进 `数据\本地配置.json` 而非改 config.json。）
3. **会话密钥**：首次启动自动生成 `数据\管理员密钥.json`（只含 cookie 签名密钥，不存业务口令）。
4. **看板账号（v8.0·v8.6 多 BU）**：`数据\看板账号.json`（不进 git）。缺文件时服务自动 seed 默认表（管理员账号 `lushasha` 初始密码 `kanban2026`；查看账号初始 `8888`）。也可参考 `docs\看板账号样例.json` 手写后放到 `数据\`。**权限**=管理员（进 `/admin`）/整体（看全部）/按 BU（v8.6：一个账号可绑**一组** BU，只看所绑 BU；一 BU 也可挂多账号）。旧「权限=单个 BU 名」账号完全兼容、编辑时自动迁移为多 BU 类型。
5. **账号初始化与改密**：
   - 部署后用 `lushasha` + 初始密码登录 `/admin` →「设置 → 👥账号与权限」：加号/改权限/改密码（明文默认打码·点👁显示；黄底=仍是初始密码）。
   - 看的人登录看板后右上「🔑密码」可自改（弹窗会提示「密码管理员可见，请勿使用你在其他地方用的密码」）；管理员端立即可见新密码。
   - **上线前把所有初始密码改掉再发账号。**
6. **BU 数据归属**：管理员端「设置 → 🏢BU 数据归属」维护 BU 名、负责人备注、销售名单（与登录账号无关）。真实配置存 `数据\BU配置.json`，不进 git。公共费用分摊/手填 BU 口径未定前页面会明确标注待补。

## 三、首次取数（v8.7.5 新顺序：先起服务、界面填好账号和路径，再点「立即更新」）

> ⚠ 全新机器上（`数据\` 只有 README）直接 `python run.py` 会报"5 处缺文件，不生成报表"——**属正常**（数据还没抓、账号还没填），不是坏了。正确顺序：
1. 先按「四、起服务」用 `看门狗启动.bat` 把服务拉起来（空机器能正常起服务）。
2. 管理端「设置→🔑智云账号·台账路径」填**智云账号密码**并保存（台账路径已预填默认值，核对即可）。
3. 管理端顶栏点 **「立即更新」** = 首次取数。预期：收单台账 fetched + 智云 下单/回款/项目明细 fetched（**内部译员在换专用账号前是兜底/体检黄，属正常**）、页面出数。约 2~3 分钟（在线抓 80 秒 + 32 个周期计算约 1 分钟）。手填与调整表没数只是警告（手填项按 0 计，后续补真数）。
之后命令行 `python run.py` 就能正常跑了（数据已在），供排查用。

## 四、起服务 + 定时更新

1. **起服务**：双击 **`看门狗启动.bat`**（推荐，内部循环跑 `run.py --serve`，支持管理端一键更新后自动重启）；或 `启动看板服务.bat`（普通版，不带自动重启）。
   - 用户端（老板看）：`http://<本机IP>:8018/`；管理员端：`http://<本机IP>:8018/admin`
   - 同 WiFi 手机可直接访问；端口可用环境变量 `KANBAN_PORT` 改。
   - 防火墙首次询问选"专用网络允许"；域环境可能需 IT 开 8018 入站规则。
   - **一键更新前提**：本项目须是 `git clone` 部署（有 `.git` + 远端）、且用 `看门狗启动.bat` 起服务，管理端「设置→版本与更新日志→检查更新」才能一键更新（安全快进拉取 `git pull --ff-only` + 看门狗按退出码 42 重启）。**对标哪个远端由 `config.json` 的 `update_remote` 决定**（默认 `origin`；从 Gitee clone 时 origin 即 Gitee，默认就对标 Gitee）。护栏：非仓库/工作区有改动/本地与远端分叉时不给自动更新（提示人工处理）。
2. **每日自动更新**：右键**以管理员身份**运行 `注册每日更新.bat`（按**合并配置**（config.json + `数据/本地配置.json` 覆盖层）的 `schedule_times` 每个时间点各注册一个计划任务，默认只有 09:30 跑 `run.py --scheduled`；多时间点=主任务 `经营驾驶舱每日更新` + `_2.._n`。⚠ 748d13b 及更早版本的 .bat 只读 config.json、读不到界面里改的时间点（F-02），升级后需重跑一次本 .bat）。之后改/增删时间点在**管理员端 → 设置 → 自动更新**里加减时间点即可（保存会自动同步计划任务）；**若增删时间点后 `schtasks /Query` 没看到对应任务变化（多半是服务进程没管理员权限建/删任务），以管理员身份重跑一次 `注册每日更新.bat` 即可**。
3. **开机自启（建议）**：把 **`看门狗启动.bat`** 快捷方式放进 `shell:startup`（Win+R 输入），重启后服务自动拉起，且支持一键更新后自动重启。

## 五、部署机必做复验清单（Windows 特有，开发机验不了）

- [ ] 首次取数（起服务后管理端「立即更新」）：共享盘 fetched + 智云三源 fetched
- [ ] `--serve` 起来后点管理员端「立即更新」→ 服务进程内完整跑一次在线抓（**Playwright 在 serve 进程内登录，开发机未实测，必须验**）
- [ ] `注册每日更新.bat` 注册成功：`schtasks /Query /TN "经营驾驶舱每日更新"` 能查到；次日确认真跑了
- [ ] 管理员端设置页加/删「更新时间点」→ 计划任务同步（`schtasks /Query | findstr 经营驾驶舱每日更新` 复核主任务+`_2.._n`；没同步则重跑 bat）
- [ ] **一键更新（用 `看门狗启动.bat` 起）**：管理端「设置→检查更新」能查到远端状态；有新版本时「一键更新并重启」→ `git pull --ff-only` 成功 →（**依赖变了会自动 `pip install`，默认走国内清华 PyPI 镜像（config.pip_mirror），无需翻墙**）→ 服务以码 42 退出 + 看门狗用新代码拉起 + 页面自动刷新（**开发机无法验，部署机务必验一次**）；无新版本时提示已最新；工作区脏/分叉时拒绝并提示
- [ ] **坏版本自愈（v8.7.3·可选验）**：若某次更新后新代码启动即崩，看门狗应据 `.update_rollback` 标记**自动回滚上一版本一次**再起（连崩 5 次仍走停下报警）；装依赖失败时一键更新应自动回滚拉取、返回失败不重启
- [ ] 手机（同 WiFi）打开用户端正常；管理员端登录、改一行数、撤销，全链路通
- [ ] `数据\备份\` 每天多一份 `看板_YYYYMMDD.db`，超过设置天数自动删最旧
- [ ] 账号分流：lushasha→管理端；整体权限账号→整体页带 BU 条；BU 权限账号→只看本 BU；错号 401；多 BU 账号→顶部「我的 BU」切换条**只列所绑 BU**（绝不列他 BU）
- [ ] 自改密码闭环；管理端账号卡明文👁/加删改/初始密码黄标

## 六、日常运维（财务只需要会这三件事）

| 事 | 怎么做 |
|---|---|
| 看数 | 浏览器开 `http://<IP>:8018/`（老板）或 `/admin`（财务） |
| 手动更新 | 管理员端顶栏「立即更新」（约 1~2 分钟，按钮显示进度） |
| 改设置 | 管理员端「设置」各卡就近保存：自动更新 / 备份 / 智云账号 / 账号与权限 / BU 数据归属 |

抓不到数不会报错中断——自动用上次数据继续算，顶栏体检变黄，点开看原因（不在内网 / 账号失效 / 共享盘不可达 / 权限不足）。

## 七、已知边界

- **内部译员源**需要能看「任务」表全量的账号（测试账号只有个人视图 85 行，被行数护栏挡住走人工导出文件兜底；换专用账号后自动全绿，代码零改）。
- 智云账号密码明文存部署机本地 `数据\智云配置.json`；看板登录密码明文存 `数据\看板账号.json`（均 gitignore；部署机在财务掌控内，既定设计）。
- **B3**：`--scheduled` 计划任务更新库后，`--serve` 服务的内存页面不会即刻刷新——日常以管理员端「立即更新」为准；此项在 R 系列迭代中解决。
- 页面"⬇导出"=当前所选周期的整页图片（服务端截图，约2秒；需 chromium 已装——装环境第3步已含）。
- 台账公式单元格若未在 Excel 里重新保存过，openpyxl 可能读到空——台账保持纯数值粘贴习惯。
- 机器关机期间定时任务不跑、页面无法访问。

> **巡检备忘（2026-07-14 部署机实测）**：Windows 上看板服务是**双 python 进程**属正常——`.venv\Scripts\python.exe` 是 venv 转发器（几 MB），它会拉起系统基础解释器（几百 MB）真正干活，但依赖仍从 .venv 加载（pyvenv.cfg 机制）、一键更新装依赖照常生效。巡检时别把这当异常处理。
