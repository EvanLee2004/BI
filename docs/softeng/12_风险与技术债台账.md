# 12 · 风险与技术债台账

> 产品 **v1.5.0-beta** · 诚实列「现在能接受 + 何时必须升级」。**不是 bug 清单**（真 bug 见 11 号台账）。

| # | 边界/债 | 当前可接受的理由 | 触发升级的条件 |
|---|---------|------------------|----------------|
| T1 | 账号密码**明文**存 `看板账号.json` | 内网 5 人、管理员可见明文是业务要求；实现成本低 | 人数扩大/需 SSO/审计强制哈希 |
| T2 | 智云**内部译员**依赖专用账号/行数护栏 | 权限不足账号会降级不中断管道 | 陆总给全量任务账号后关黄 |
| T3 | 预渲染 HTML **体积随周期与排名涨** | 数据量仍小、日更一次 | 打开卡顿/手机超时 |
| T4 | **SQLite 单文件**单机 | 零运维、Windows 财务机；**任务书33 已上 WAL+单事务重建+整数分+quick_check**，单机写并发可接受 | 多写并发/要 HA/要只读副本分离 → Phase3 Postgres |
| T5 | 管理端/看端 **LEGACY 内嵌常量双源** | 一键回退；测试锁字节同文 | 双源漂移事故频发 → 删内嵌只留 static |
| T6 | 登录页仍小体积**内嵌 server.py** | 体量小、非主路径 | 要统一主题构建时再抽 |
| T7 | shell 会话过期曾跳 **`/login` 无路由**（边缘） | 日常少碰 | 用户报「更新后白屏/死链」则改跳 `/` |
| T8 | 无公网、无 HTTPS 默认 | 内网信任边界 | 跨网段/合规要求 TLS |
| T9 | 手填/分摊/去税 **靠人记** | 业务就是人工口 | 缺月漏填导致决策错 → 加强强制校验/提醒 |
| T10 | 附加税费措辞 vs 陆总口头「收入×6%×12%」 | 代码用不含税基数与税法一致 | 陆总明确改基数则改 profit |
| T11 | 公共费用分摊比例 **业务细则**仍靠手填 | 机制已上线 | 陆总定 100% 分光/全年一套等再收紧校验 |
| T12 | 测试活体浏览器 **未进 CI** | 本地/部署机人工 | 再出 dTbl 类假绿 → 加 headless 冒烟 |
| T13 | `parse_date_parts` **不校验非法日历日**（如 2/30） | 坏行多半进不了合法周期；历史数据兼容 | 明昊拍板是否严校后改解析 |
| T14 | 预算表 **比率与金额共用「分」存** | 数值 round-trip 一致；schema 简单 | 比率列语义混淆时拆列 |

## 升级排序建议（明昊视角）

1. 配置类黄灯清零（销售 BU 归属、译员账号）— 业务价值高、改码少  
2. shell 401 跳转修掉 — 小改  
3. 密码哈希/飞书 SSO — 见生产化路线图 Phase3  
4. CI 真实退出码 + 冒烟 — 防假绿
