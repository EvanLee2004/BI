## 部署步骤（严格按 `docs/上线交接包/部署执行单_给Claude.md` + `docs/Ubuntu部署手册.md` 执行——57·B-2 已演练修订过,照文档做;文档与实机不符=记文档 bug,保守处理）

2. **push+取码(你自己做,明昊已授权)**:① 本地 `git push origin main && git push gitee main`——**只推 main,绝不 push --tags/其他分支**;安全检查 Claude 已于 2026-07-19 夜完成并记录(真实客户名0/内网IP仅白名单UNC/password命中全为公开豁免种子账号),你 push 前再快扫一遍 `git diff origin/main..main` 中新增部分确认无真实客户名/密码即可;② 机上 `sudo mkdir -p /opt/kanban && sudo chown lee /opt/kanban`,`git clone`(Gitee 优先,不通换 GitHub;都不通=git bundle scp 兜底)→ `git -C /opt/kanban rev-parse HEAD`==本地 main HEAD。
3. **环境**:venv+`pip install -r requirements.txt`(用 config 里 pip 镜像)+`playwright install chromium`(智云自动登录用)。
4. **数据初始化**:按部署手册建 `数据/`;config `data_dir` 指真实数据目录;**不放任何合成数据进生产**;首启走引导页逻辑。
5. **服务与自愈**:systemd 单元(看门狗 `deploy/linux/start_with_rollback.sh`,Restart=always)+`systemctl enable --now`;nginx 站点(dist 静态+API 反代:80)+`nginx -t` 后 reload;crontab:每日更新(`register_schedule.sh`)+healthcheck 每小时+备份 cron。
6. **冒烟(照执行单 D 清单)**:内网 `curl localhost/login` 200;版本显示 2.0.0-rc12;`bash deploy/healthcheck.sh` EXIT:0;`systemctl is-active kanban`=active;**重启自愈演练**:kill 主进程→看门狗 20 秒内拉起(记录证据)。
7. **凭据落位**:按安全铁律3,scp 本机 `数据/智云配置.json` 到部署机 `数据/`;确认文件权限 600、属主 lee;顺带确认 config 的收单台账共享盘路径为出厂默认(部署机在内网可直读)。**不停不等,直接进第 8 步。**
