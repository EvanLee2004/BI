# v1.5.0-beta 管理端前后端分离 · 交付说明

基准：`393c04b` / v1.4.0-beta（含精简视图下线 + 顶栏 sticky）。  
原则：**搬家不装修**（无 React / npm；管理端观感与交互对齐内嵌版）。

## 拆了什么

| 原位置 | 现位置 |
|--------|--------|
| `server.py` `_ADMIN_CONSOLE` 内联 CSS | `static/admin/admin.css` |
| 内联应用 JS | `static/admin/admin.js`（磁盘含 `__MANUAL_ITEMS__` 占位） |
| 整页骨架 | `static/admin/admin.html`（`<link>` + `<script src="/admin/app.js">`） |
| `_BOOTSTRAP_HTML` | `static/admin/bootstrap.html`（首次取数引导） |

常量 `_ADMIN_CONSOLE` / `_BOOTSTRAP_HTML` **仍保留在 server.py**，供回退。

## 运行时怎么走

1. 默认：`GET /admin`（已登录且已有数据）→ 返回 `static/admin/admin.html`  
2. 浏览器拉 `/static/admin/admin.css`、`/admin/app.js`  
3. `/admin/app.js`：读磁盘 `admin.js`，把 `__MANUAL_ITEMS__` 换成当前 `config.manual_items` JSON（纯注入）  
4. 未登录 → 登录页（仍内嵌 `_LOGIN_HTML`，未强制拆）  
5. 无 `admin_html` 缓存 → 引导页（优先 static bootstrap）

## 回退

- 环境变量：`KANBAN_LEGACY_INLINE=1` → `/admin` 直出内嵌 `_state["admin_html"]`（与 v1.4 看端 shell 同一开关语义）  
- 代码回退：`git checkout 393c04b` 后重启看门狗  
- unittest / pytest：默认仍内嵌（`unittest` 在 `sys.modules` 时不走 static），旧测试不破  

## 未改

- 依赖：`requirements.txt`、五个 `.bat`
