# 批次 1 · 陆总 #8 排名行 1~12 月下钻（2026-07-16）

## 需求
下单与回款板块（整体页 + BU 页）排名行点击后，展开/弹窗显示该主体 1~12 月各月下单、回款金额。与 `.rk-full`「其余」同一套 `rkModal` 交互。

## 实现（任务书34 归一化后）

| 层 | 做法 |
|----|------|
| `profit.build_period` | 算 `rankings_monthly`：只挂排名出现的销售/客户（top+其余全量），年=期间起算年 |
| `profit.compute_name_month_totals` | 纯数矩阵 name×12 月 |
| `render.attach_monthly_to_dual` | 显示串 + 宽度% 写入**页面级字典**；行上只挂 `mkey`（`年\|销售\|名` / `年\|客户\|名`） |
| `api_v1.rankings_view_for_period` | items/full_items 只带 `mkey`；单周期可带 `monthly_data`；多周期经 `monthly_store` 去重 |
| `api_v1.build_*_cockpit_views` | 根上 **一份** `rankings_monthly_data`（全周期共享，不按 period 重复） |
| `rankings.js` | 行 `data-mkey`；`paintRankingMonthly` 查 `__rkMonthlyData` / `#rkMonthlyData` 拼 DOM |
| `page.js` | 组装排名时注入一次 `#rkMonthlyData` 脚本，周期 `.pv` 不再嵌月度 JSON |
| `cockpit.js` / `cockpit-bu.js` | 点 `.rk-entity[data-mkey]` → 弹窗 1~12 月；「其余」优先本地 `.rk-full` |

### 为什么改（任务书34 blocker）
旧实现把 12 月 JSON 以 `data-monthly="[...]"` 嵌在**每一行**，且每个周期页签各拷一份 → 整页/fragments 膨胀到十余 MB（手机不可接受）。  
**修法**：去重后的主体月度显示串集中在页面级结构；行上仅短引用；点击本地查表（零新 API、前端零金额运算）。

### 体量（golden 测试数据实测 2026-07-16）
- `/api/v1/cockpit/fragments`（client pack JSON）≈ **2.85 MB**（≤ 3MB 门槛）
- 行级 `monthly` 大数组：**0**；`mkey` 覆盖全部排名 items/full_items
- 详见 `docs/20260716_任务书34_月度下钻payload.md`

## 铁律
- 前端零金额运算（宽度/金额 Python 预算；JS 只 `toFixed` 拼 CSS / 拼已有显示串）
- BU 零新跨界 API；整体页也不加新 API
- 不倾倒全库：monthly 键 ⊆ 排名 full_items 名单
- 弹窗 DOM/文案与改前一致（`dual-month` 双血条 + `· 1~12 月下单/回款` 标题）
