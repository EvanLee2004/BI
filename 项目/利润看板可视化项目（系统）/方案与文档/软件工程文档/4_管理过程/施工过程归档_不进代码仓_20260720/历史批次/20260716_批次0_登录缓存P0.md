# 批次 0 · 登录页 HTTP 缓存 P0（2026-07-16）

## 现象
登录成功后 `location.replace` 跳回同 URL，浏览器仍显示登录页（不刷新则永远卡住）。

## 根因
未登录时 `/`、`/admin`、`/bu/*` 用 `FileResponse` 回登录页 HTML，响应**未带** `Cache-Control`。浏览器把「该 URL = 登录页」缓存；登录成功后同 URL 导航直接吃磁盘/内存缓存、不再问服务器。

证据（Claude 实锤，本批 TestClient 复现路径）：
- 默认 `fetch('/admin')` 可命中缓存登录页；`cache:'no-store'` 返回控制台。
- `curl`（无浏览器缓存）一直正确。

## 修法
| 路由/响应 | 处理 |
|-----------|------|
| `_view_login_file` / `_admin_login_file` | `FileResponse` + `Cache-Control: no-store` |
| `_main_shell` / `_bu_shell` | 同上 |
| `/admin` 控制台 / bootstrap 引导页 | `HTMLResponse` + no-store |
| `/api/history/{day}` 快照 HTML | 同上 |
| `/admin/app.js` | 已有 no-store（先例，未改行为） |
| `/static/*` css/js/图 | **不动**（可缓存） |

实现：`src/server.py` 增加 `_NO_STORE` / `_html_doc` / `_file_html_doc`，文档路由统一走它们。

## 401 → 登录
- `static/shell.html`、`static/shell-bu.html`：fragments 401 → `location.replace('/login')`（已有，本批守卫）。
- `static/admin/admin.js`：API 401 → `/admin`（已有）。
