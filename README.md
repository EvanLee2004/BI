# 甲骨易智能经营罗盘（利润看板）

> 一套**轻量自建 BI**：每天自动从业务系统抓数 + 读财务台账，算到**税前利润**，出科技风暗色（可切浅色）、手机可看的经营驾驶舱；管理员可在网页上改数、手填、看历史。  
> 零重型 BI 依赖——**Python + SQLite + FastAPI + 手写 SVG**；**无 React / 无 npm**。

## 这是什么

给管理层看的**经营利润驾驶舱**。业务背景：语言服务公司要一个「脱离 Excel、每天自动更新、手机能看」的实时利润视图——收入、成本、五类期间费用、税前利润；能按任意周期切、下钻到费用明细细类、看下单/回款排名与收入结构。

**同一端口、账号分流**（不是多个链接硬分权限）：

| 端 | 入口 | 谁用 | 能干什么 |
|---|---|---|---|
| 整体用户端 | `/` · 权限=「整体」 | 管理层 | 只读全公司：KPI、利润表、结构、排名；顶栏进各 BU；切周期；整页 PNG 导出；自改密码 |
| BU 用户端 | `/` · 权限=某 BU（可绑多 BU） | 业务负责人 | 登录后直接本 BU 页（完整利润表，按销售名单过滤、**跨 BU 不泄漏**）；多 BU 账号顶栏只列自己绑定的 |
| 按时间段看 | 整体页板块日期条 | 已登录整体/管理员 | 日期条跟顶；任意起止日查询排名；「其余」全量弹窗（`/api/daily`，BU 会话 401） |
| 管理员端 | `/admin` | 财务经理 / 管理员 | 改明细（留痕可撤销）、手填/分摊/去税、预算、异常处理、账号与销售归属、一键更新、设置 |

**核心取舍**：前端**零金额运算**——年/季/月/任意月区间（30+ 周期）的全部板块在 Python 侧预渲染成 HTML 块，前端只做显示切换。口径唯一、可测、可回归对账。

## 系统架构

![系统架构图 v1.5](docs/设计图/02_概要设计_系统架构图.svg)

> 上图为 **架构图 v1.5**（2026-07-15），对齐产品 **v1.4.0-beta**：抓数已建成、账号分流与 BU 页已上线、展示层拆出 `static/` + `/api/v1/*`。  
> 第一次读项目建议先看大白话版：[小白版运行逻辑图](docs/设计图/附_小白版运行逻辑图.svg)。

### 五层单向数据流

```
① 抓数层   智云四源自动登录在线抓 + 收单台账 SMB + 管理员表单手填
   └▶〔进料口〕数据/ 目录（6 源文件 + 手填库 + 账号/BU/智云配置）——上下游唯一接缝
② 清洗层   ingest：读原始 → 规范化（行哈希定位键）→ 重放人工调整（过期校验）
③ 存储层   SQLite 单文件：std_ 标准表 + adj_/manual_/meta_ 人工表
④ 计算层   profit：只读库算 summary（32 周期矩阵，纯函数；BU 过滤 / 按月分摊 / 去税）
⑤ 展示层   FastAPI：账号分流页面 + static/ 外置资源 + /api/v1 JSON（管理端 HTML 仍在 server.py）
```

### 三条架构契约

1. **进料口是唯一接缝**：换数据源 / 换抓取方式只动①和 readers，②以下不动。
2. **数据库是后端私有资产**：浏览器只经 HTTP 跟 FastAPI 要数据，从不直接碰库。
3. **抓数失败永不中断**：登录失败 / token 失效 / 行数异常一律三态降级（沿用本地副本 + 体检黄），管道照跑。

### v1.4 展示层怎么拆的

| 组件 | 作用 |
|---|---|
| `static/css/theme.css` | 主题与样式（改 UI 不必动 Python 算账） |
| `static/js/cockpit.js` / `cockpit-bu.js` | 整体页 / BU 页交互（只切显示，零金额运算） |
| `static/shell.html` | 登录后壳：`fetch /api/v1/cockpit/view` 灌入同源 HTML |
| `GET /api/v1/cockpit` | 驾驶舱 JSON（`numbers` 与 `golden/baseline_numbers.json` 全等） |
| `GET /api/v1/cockpit/view` | 像素级 HTML（仍由 `render` 生成，观感对齐 v1.3.1） |

回退整页内联 HTML：`KANBAN_LEGACY_INLINE=1 python run.py --serve`（或 `git reset --hard efacf8c` 回到 v1.3.1）。

每日更新 / 管理员改数的完整时序见 [关键流程时序图](docs/设计图/03_详细设计_关键流程时序图.svg)；表结构见 [数据库 ER 图](docs/设计图/03_详细设计_数据库ER图.svg)。

## 6 个数据源

| 源 | 供什么 | 类型 | 获取方式 |
|---|---|---|---|
| 项目明细（智云） | 收入（交付额÷1.06）+ 系统直接成本 | 系统 | 自动登录在线抓（只抓当年） |
| 任务·内部译员（智云） | 内部译员成本（从成本中减出） | 系统 | 自动抓 + 行数门槛护栏（待专用账号） |
| 下单（智云） | 下单额 + 部门/销售排名 | 系统 | 自动抓 |
| 回款记录（智云） | 到账额 + 客户排名 | 系统 | 自动抓 |
| 收单台账（Excel） | 市场/管理/固定运营/技术服务/财务五类费用 | 台账 | SMB 共享盘抓（不可达走本地副本） |
| 手填与调整 | 人力成本、生产成本手填、财务费用补充、其他损益 | 手填 | 管理员端表单（**当月未填=0**，体检提示缺月） |

仓库**不含任何业务数据**：`数据/` 整目录被 .gitignore 挡住（库 / 凭据 / xlsx / 备份均不进 git），真实数据只存在部署机本地。

## 计算业务逻辑

管理利润表口径（财务经理定稿，`config.json` 是税率与费用分类的唯一配置源）：

```
收入(不含税)  = 交付额 ÷ 1.06                          ← 智云项目明细
生产成本      = 系统直接成本 − 系统内部译员成本
              + 手填(PM/VM/实际内部译员/税费损失/技术流量/其他)
              − 手填「直接成本增值税」(默认 0)
毛利          = 收入 − 生产成本
营销费用      = 营销人力成本(手填) + 市场费用(台账)
管理费用      = 管理人力成本(手填) + 管理费用(台账)
固定运营费用  = 台账
研发费用      = 研发人力成本(手填) + 技术服务费(台账)
财务费用      = 财务费用(台账) + 财务费用补充(手填)
附加税费      = 增值税额 × 12%（增值税 = 不含税收入 × 6%）
税前利润      = 毛利 − 五类费用 − 附加税费 + 其他损益(手填,默认0)
```

几个关键机制：

- **周期矩阵**：全年 / Q1~Q4 / 各月 / **任意连续月区间**，全部独立算好预渲染；区间数字=成员月加和（回归红线锁死）。
- **手填当月未填=0**：不再沿用上月；缺月体检提示。年/季=期间内各月之和。
- **人工调整可重放**：改值/剔除记成「调整指令」（不改原始），每次重抓后自动重放；源头变了→「过期疑似」标黄，定位键失配→计数提醒，**不会静默失效**。
- **公共费用按月分摊**：管理端填每月 BU 比例（合计可&lt;100%，残留留公司层）；比例沿用最近填写月。
- **费用去税率**：台账费用可按类别填去税率（空=不去税）；源头统一换算，全链路一致。
- **费用归集**：收单台账按「对应报表大类」白名单路由；未分类不计入（有提示）；按预算归属部门/利润中心两视角合计守恒。
- **数据体检**：每轮更新判绿/黄/红，页面徽章 + 管理端原因区。

## 页面功能导览

**用户端四段骨架**：

1. **基本情况**：5 张 KPI 卡（收入/毛利/税前利润/下单/回款）+ 全年峰值 + 交付占下单 / 已交付未回款等扫读提示。
2. **经营利润**：收入成本毛利趋势 → 管理利润表（点大类→抽屉两级下钻，含「分摊自公共」）→ 期间费用构成（大类环形｜类别｜业务 BU）→ 回款按月（全年柱 + 选中周期高亮）。
3. **收入与毛利结构**：收入·按客户 / 按销售 + 集中度；「其余」全量弹窗（`/api/profit_ranking`）；确认口径=交付额÷1.06、毛利=收入−项目成本（项目直接毛利）。
4. **资金与回款**：回款情况 + 下单/回款排名（按部门·销售·客户·BU）；日期条常显跟顶。

全局：顶部日历式周期选择器、暗/浅主题（跨窗同步）、数字标来源（智云/台账/手填）、体检徽章、**⬇ 导出 = 当前周期整页 2× PNG**（服务端 Playwright，约 2 秒）。

**管理员端**：数据总览 / 明细台账 / 手填与调整（含分摊比例、去税率）/ 年度预算 / 异常处理（总览、调整台账、下单未填部门、费用未分类、历史快照、操作记录）/ 设置（自动更新多时段、备份、智云账号、账号与权限、销售归属、数据路径、版本与一键更新）。

**账号与 BU**：`数据/看板账号.json`（管理员 / 整体 / 按 BU；可绑一组 BU）+ `数据/BU配置.json`（BU 名 + 销售名单 = 子页数字口径）。样例见 `docs/看板账号样例.json`、`docs/BU配置样例.json`。两文件不进 git。

**一键更新 + 看门狗**：设置页比对远端 → `git pull --ff-only` → 依赖变了自动 `pip install`（默认清华镜像）→ 退出码 42 看门狗重启；坏版本启动即崩自动回滚一次。需用 `看门狗启动.bat` 起服务。

## 快速开始

```bash
git clone https://github.com/EvanLee2004/BI.git && cd BI
python -m venv .venv && .venv/bin/pip install -r requirements.txt
.venv/bin/playwright install chromium        # 图片导出与智云自动登录用

# 把 6 个数据文件放进 数据/（文件名对照见 数据/README.md；仓库不带数据）
python run.py             # 更新一次：抓数→建库→算→出 output/经营驾驶舱.html
python run.py --serve     # 起内网服务（端口见 config，KANBAN_PORT 可覆盖）
python run.py --scheduled # 给 Windows 计划任务调用（每日自动更新）

KANBAN_OFFLINE=1 sh tests/run_verify.sh   # 一键验证（离线，不碰网络）
```

- 账号在 `数据/看板账号.json`（明文、不进 git；缺文件自动 seed）。管理员默认 `lushasha` / `kanban2026`；查看账号初始 `8888`——**上线前改掉**。
- 同一入口 `/`：账号+密码按权限分流（管理员→`/admin`、整体→整体页、BU→本 BU）。
- 智云账号密码在管理端·设置页填，保存即用。
- 完整装机见 [docs/Windows部署手册.md](docs/Windows部署手册.md)。
- 测试/正式数据切换只改 `config.json` 的 `data_dir`，代码零改动。

## 质量保障与铁律

- **回归红线**：`tests/regress_db_vs_files.py`——从库算与从文件直算，32 个周期全部数字逐字段一分不差。
- **golden 全等（v1.4）**：`/api/v1/cockpit` 的 `numbers` 与 `golden/baseline_numbers.json` 全等。
- **守恒红线**：费用分组合计==台账白名单合计；区间==成员月之和。
- **前端不算数守卫**：产物 HTML 出现 `toFixed` / `parseFloat` 即失败。
- **验证只认真实退出码**：绝不套 `| tail` / `| head` 判绿（会吞失败码）。
- 开发铁律全文见 [CLAUDE.md](CLAUDE.md)（智云 xlsx 绝不 `read_only`、列按表头找、必需列缺失即报错、BU 严格隔离、配置分离不脏 config.json……）。

每个改动批次：补测试 → `run_verify.sh` 全绿 → 同步 CHANGELOG → commit + push。

## 版本与发布安全

- 产品阶段：0.9 试运行 → **1.x-beta 公测 Beta（现 `1.4.0-beta`）** → 1.0 正式版（去 `-beta`）。
- **`main` 是唯一开发与发布分支**；`archive-*` 为含历史数据的本机档案，永不推送、绝不 `push --tags`。
- push 前安检：`git log origin/main..main --name-only` 逐 commit 核——无 xlsx/csv/db、无真实金额/客户名/内网 IP/账号密码。

完整变更记录在项目本地文档库 `方案与文档/软件工程文档/4_管理过程/CHANGELOG.md`。
